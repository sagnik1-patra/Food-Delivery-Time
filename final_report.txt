
Food Delivery Time Prediction using CNN and Model Validation

Author:
Sagnik Patra

Objective:
The goal of this project is to predict whether a food delivery will be Fast or Delayed
based on customer location, restaurant location, weather conditions, traffic conditions,
delivery person experience, order priority, ratings, vehicle type, distance, and other
delivery-related features.

Dataset:
Food_Delivery_Time_Prediction.csv

Dataset Shape:
(200, 23)

Target Variable:
The target variable was created from Delivery_Time.

Fast:
Delivery_Time <= Median Delivery_Time

Delayed:
Delivery_Time > Median Delivery_Time

Median Delivery Time:
72.775

Preprocessing:
1. Missing numerical values were handled using median imputation.
2. Missing categorical values were handled using most frequent imputation.
3. Categorical variables were encoded using One-Hot Encoding.
4. Numerical variables were normalized using StandardScaler.
5. Location coordinates were extracted from customer and restaurant locations.

Feature Engineering:
1. Customer latitude and longitude were extracted.
2. Restaurant latitude and longitude were extracted.
3. Haversine distance was calculated.
4. Order hour was extracted.
5. Rush hour feature was created.
6. Delivery status was created as Fast or Delayed.

CNN Method:
The dataset was tabular, so the processed feature vector was converted into an
image-like square matrix. This allowed the CNN model to process the features using
Conv2D layers.

CNN Architecture:
1. Conv2D Layer
2. Batch Normalization
3. MaxPooling2D
4. Conv2D Layer
5. Batch Normalization
6. MaxPooling2D
7. Flatten Layer
8. Dense Layer
9. Dropout Layer
10. Sigmoid Output Layer
