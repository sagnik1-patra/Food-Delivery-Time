
Food Delivery Time Prediction using CNN

Objective:
The goal of this project is to predict whether a food delivery will be Fast or Delayed
based on customer location, restaurant location, weather, traffic conditions, delivery
person experience, order priority, vehicle type, ratings, cost, tip amount, and time-based features.

Dataset:
Input File:
C:\Users\NXTWAVE\Downloads\Food Delivery Time\Food_Delivery_Time_Prediction.csv

Dataset Shape:
(200, 23)

Target Creation:
The Delivery_Time column was converted into a binary classification target.
Deliveries less than or equal to the median delivery time were labelled as Fast.
Deliveries greater than the median delivery time were labelled as Delayed.

Median Delivery Time:
72.775

Feature Engineering:
1. Customer latitude and longitude were extracted from Customer_Location.
2. Restaurant latitude and longitude were extracted from Restaurant_Location.
3. Haversine distance was calculated between customer and restaurant.
4. Rush hour feature was created using Order_Time.
5. Order hour was extracted from Order_Time.
6. Categorical features were encoded using One-Hot Encoding.
7. Numerical features were scaled using StandardScaler.
8. Missing numerical values were handled using median imputation.
9. Missing categorical values were handled using most frequent imputation.

CNN Method:
Since the dataset is tabular and does not contain real route/map images, the processed
feature vector was converted into an image-like matrix. This allowed CNN layers to process
the structured delivery features.
